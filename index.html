Yo.js

Yo.js is an open source framework to modularize your javascript and css code.

Welcome to Yo.js.

Yo.js is an open source framework to structure your javascript code base by following your own design pattern. It provides a maintainable code base for developers to focus on optimizing and develop a clean solution. It also comes with add on templating utility library to structure your DOM in an organised way.

The project is hosted on GitHub. You can report bugs and discuss features on the GitHub issues page. Yo.js is available for use under the MIT software license.

Introduction

When working on large scale web applications which involves lots of javascript and css code. It really becomes pain for developers to organize the code in there desired manner. As developer you would want to follow your own design pattern.

But while following your own design pattern we would end up writing whole chunk of code in multiple js files each time while adding a js file you need to go ahead and include the js file into your html. At certain point it becomes the tedious task for developers.

Yo.js tries to solve your problem by tracking and synchronously including all js and css files. It provides flexibility for developers to have their own namespace mapped to there respective js files.

Yo.js is currently in beta version.

Tested and supported browsers IE 9+, Mozilla, Chrome.

Usage & API's

  1. Setup configuration to include js and css files

        yo.core.config({

       // Sets the root folder as the base directory for all future relative paths
       baseUrl: "",

       // Set the namespace for the application
       namespace: "App",

       // 3rd party script alias names (Easier to type "jquery" than "libs/jquery, etc")
       path: {
           model: "js/app/model",
           view: "js/app/view",
           controller: "js/app/controller",
           libs: "js/libs",
           styles: "css"
       },

       //Load JS files
       loadScript: {
           libs: ["jquery"],
           model: ["sample"],
           view: ["sample"],
           controller: ["sample"]
       },

       //Load CSS & LESS file
       loadCSS: {
           styles: ["style", "style.less"]
       }

    }, function () {
       //Callback function once files are loaded
       console.log(App, "Namespace is defined"); 
       //(Define Class) App.model.sample, App.view.sample, App.controller.sample 
    });

  • baseURL - Sets the root folder as the base directory for all future relative paths. Modify to your respective root path.

  • namespace - Defines the namespace for your application.

  • path - Define your path name or alias name for respective folders.

  • loadScript - Include the js files in its hierarchy. The property name is mapped to the path and the namespace such as (App.Model, App.View, App.Controller). We can also provide the extension but its not mandatory.

  • loadCSS - Include the css and less files in its hierarchy.

Yo Templating Library Usage

Yo.js comprises of stand alone light weight templating library similar to moustache or handlebar with simple and easy to feature to keep your DOM away from your javascript code.

1. Inserting Div Section from HTML Template

a) Sample HTML Template

<div class="test"> Show this div section. </div>
<div class="test2"> Not to be displayed </div>

b) Add test class from the above template

 var insert = yo.utils.template.add({
       ele : ".test", // div section to be appended
       root : ".hero", // parent DOM reference to be append / default body
       template : "test.html", // template file to be included
       mode : "append" // default is "append" / "insert" 
}, function () {
    console.log("callback executed once div is appended");
});

2. Bind JSON data to the template

a) Sample HTML template

<div id="test3"><strong>{name}</strong> is working at {company} for {years}.</div>

NOTE : Curly braces "{ }" refers to the variable

b) Bind JSON data to the template

var jsonData = {
    name : "John",
    company: "SocioMobilePlus",
    years: 1
};

var test3 = yo.utils.template.add({
    ele: "#test3",
    root: ".hero",
    template: "test.html", // template file to be included 
        mode : "insert", // inserts the div or container 
    data: {
        name: jsonData.name,
        company: jsonData.company,
        years: jsonData.years
    }
}, function () {
    console.log("callback executed once div is appended / inserted");
});

3. Bind array of data

a) Sample HTML Template

<div class="test4"> {slno}.{name} is present today.</div>

b) Bind array of data from service layer

var namesArray = [{slno: 1, name: "Ajain"}, {slno: 2, name: "Vivek"}];

var test5 = yo.utils.template.add({
    ele : ".test4",
    root : ".hero",
    template: "test.html",
    data: namesArray // Bind array of data
}, function () {
    console.log("callback executed once div is appended / inserted");
});

4. Remove the container from the DOM

Remove method removes the element or container from DOM but not from the cache

yo.utils.template.remove({
    uid: test.uid or "#ele / .class" // uid refers to the unique dom id while adding the element on DOM 
}, function () {
        console.log("callback executed once div is removed"); 
});

**5. Destroy the container from the DOM **

Destroy method removes the element or container from DOM and from the cache memory

yo.utils.template.destroy({
    uid: test.uid or "#ele / .class" // uid refers to the unique dom id while adding the element on DOM 
}, function () {
        console.log("callback executed once div is destroyed"); 
});

6. Fetch Cached DOM

Cached DOM elements can be later used for adding elements on to the DOM. Cached elements drastically boosts the performance to add/ remove container.

console.log(yo.utils.template.cache());
